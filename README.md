# 赚钱信息分析知识库

一套结构化的赚钱信息分析系统。不是用来"收集项目"，而是用来**看懂项目本质、拆解赚钱结构、形成判断力**。

核心定义：任何赚钱项目，本质都是——用某种能力或资源，满足某类人的某种需求，再通过流量、转化和交付，把需求变成可重复的收入。

## 使用方法

### 准备工作

将你看到的赚钱案例信息（文章、截图文字、视频要点等）粘贴或保存到 `原始素材/` 目录。

### 两种分析模式

在 Cursor 中新建对话，将赚钱信息粘贴给 AI，然后选择模式：

#### 模式一：提问模式

告诉 AI "帮我提问" 或 "我想自己找答案"。

AI 会：
1. 判断信息属于哪种类型（项目/逻辑/手段/话术/风险玩法）
2. 提取已知信息
3. 针对缺失维度生成结构化问题清单（标注分析维度和优先级）
4. 给出调研建议（去哪里找答案）

适合：想锻炼自己分析能力、深度调研一个项目时使用。

#### 模式二：分析模式

告诉 AI "帮我分析" 或 "你来分析"。

AI 会：
1. 按 9 步标准流程逐步分析
2. 输出完整的分析卡（Markdown 格式）
3. 对信息不足的维度标注需要补充什么
4. 给出项目层级拆解、母型归类、放大路径建议

适合：快速评估一个项目、批量分析多个案例时使用。

### 保存分析结果

分析完成后，将分析卡保存到 `案例分析/` 目录，建议命名格式：`项目名称.md`

## 分析框架速查

**9 大核心问题**：赚什么钱 → 卖给谁 → 为什么买 → 交付什么 → 怎么获客 → 最难在哪 → 壁垒是什么 → 风险是什么 → 能沉淀什么

**5 种信息类型**：具体项目 / 赚钱逻辑 / 放大手段 / 包装话术 / 高风险玩法

**9 种项目母型**：技能接单 / 内容课程 / 电商卖货 / 流量变现 / IP信任 / 信息差套利 / 渠道撮合 / 品牌溢价 / 平台规模

**6 个判断标准**：需求真实 / 获客可持续 / 交付可复制 / 利润够厚 / 风险可控 / 能沉淀资产

**总纲 8 句话**：
1. 看项目，不看表面动作
2. 看它赚什么钱
3. 看它卖给谁
4. 看用户为什么买
5. 看流量从哪来
6. 看交付怎么做
7. 看能不能放大
8. 看风险和资产沉淀，再判断适不适合自己
