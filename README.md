# 个人赚钱成长操作系统 — Cursor 智能环境

基于《个人赚钱成长操作系统》方法论，在 Cursor 中搭建的可交互赚钱成长陪跑系统。

---

## 使用方式

你只需要用自然语言对 AI 说话，系统会自动识别你所处的阶段，调用对应的分析框架和模板。

### 六个核心命令

| 你说的话 | 系统做什么 | 适用场景 |
|---|---|---|
| **"我有个想法"** + 描述 | 先提炼信息 → 扩展维度 → 做判断 → 给下一步建议 | 脑子里有个模糊想法，想看看值不值得做 |
| **"分析这个"** / **"帮我分析"** + 案例内容 | 按9步分析流程做完整项目拆解，输出分析卡 | 看到一个赚钱案例/文章/项目，想深入理解 |
| **"帮我筛选"** / **"比一比"** | 对多个候选项目做双层筛选（执行可行性+商业价值），给出评级 | 手上有几个方向，不知道先做哪个 |
| **"帮我设计闭环"** / **"我想验证"** | 设计最小闭环（MVP）+ 拆解执行计划 + 时间安排建议 | 确定了方向，要开始真正做 |
| **"我卡住了"** / **"没效果"** / **"出问题了"** | 六层系统排查（项目/流量/转化/交付/执行/认知偏差），定位问题并给单变量调整建议 | 做了一段时间，遇到卡点或没有反馈 |
| **"帮我复盘"** / **"这周总结"** | 复盘七步 + 高价值动作识别 + 九类资产沉淀检查 | 一周/一轮验证结束，需要总结和决策 |

### 典型使用流程

```
1. 看到一个赚钱案例 → 说"分析这个" → 得到完整分析卡
2. 觉得值得做 → 说"帮我筛选" → 和其他候选比较，确认优先级
3. 决定验证 → 说"帮我设计闭环" → 得到 MVP 设计 + 执行拆解
4. 执行中遇到问题 → 说"我卡住了" → 六层排查定位问题
5. 一周结束 → 说"帮我复盘" → 总结 + 资产检查 + 下周决策
6. 循环迭代
```

每个阶段完成后，AI 会自动提示你可以说什么进入下一阶段。

## 系统结构

### Rules（自动生效的判断准则）

| Rule | 文件 | 何时生效 |
|---|---|---|
| 赚钱成长总纲 | `.cursor/rules/赚钱成长总纲.mdc` | 始终生效，识别阶段并路由 |
| 赚钱分析助手 | `.cursor/rules/赚钱分析助手.mdc` | 用户提供赚钱案例/信息时 |
| 执行教练 | `.cursor/rules/执行教练.mdc` | 用户提到执行/卡点/问题时 |
| 资产沉淀守护者 | `.cursor/rules/资产沉淀守护者.mdc` | 用户做复盘/总结时 |

### Skills（专项能力包）

| Skill | 目录 | 用途 |
|---|---|---|
| 项目深度分析 | `.cursor/skills/project-analysis/` | 9步分析法，输出完整分析卡 |
| 项目筛选比较 | `.cursor/skills/project-screening/` | 双层筛选 + 多项目横向比较 |
| 最小闭环设计 | `.cursor/skills/mvp-design/` | MVP 设计 + 执行拆解 + 时间安排 |
| 问题诊断排查 | `.cursor/skills/problem-diagnosis/` | 六层排查 + 单变量调整 |
| 复盘与资产沉淀 | `.cursor/skills/review-and-assets/` | 复盘七步 + 高价值动作 + 九类资产检查 |

### 模板（`.cursor/模板/`）

17 个结构化模板，覆盖从案例收集到资产沉淀的全流程。使用顺序见 `模板15：整套模板的最简使用顺序.md`。

### 输出目录

- `案例分析/` — 每次分析产出的结构化文件，命名格式：`YYYY-MM-DD-项目名称.md`

## 核心理念

> 项目是表层，系统才是底层。

这套系统不是帮你"找项目"，而是帮你建立一套能分析项目、筛选项目、验证项目、沉淀项目的方法。

- **判断层**解决"方向对不对"
- **执行层**解决"事情动不动"
- **积累层**解决"做完值不值"

三层打通，赚钱成长才能进入正循环。
