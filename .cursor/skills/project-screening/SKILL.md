# 项目筛选比较

基于《个人赚钱成长操作系统》第1篇第7-8章。详细原则见 [reference.md](reference.md)。

## 核心原则

> 「你能做，不等于值得做。」
> 「最理性的做法不是问这个项目好不好，而是问：对于我当前阶段，这个项目有没有资格进入接下来30天的验证名单。」
> 「前期最重要的不是"绝不选错"，而是"建立一套能通过反馈修正选择的机制"。」

## 筛选流程

### 第1步：确认候选项目

确认用户要比较哪些项目。如果用户只有一个项目，直接做双层筛选；如果有多个，做横向比较。

### 第2步：第一层筛选 — 执行可行性

对每个项目评估5个维度（1-10分）：

| 维度 | 核心问题 |
|---|---|
| 启动难度 | 我现在能不能开始？需要准备很久吗？ |
| 反馈速度 | 做了之后多快能看到反应？ |
| 成本 | 时间/金钱/认知/情绪成本高不高？ |
| 与当前能力距离 | 离我现在的能力近不近？ |
| 可持续性 | 我能不能稳定做30天？ |

输出执行可行性总评、最大优势、最大障碍。

### 第3步：第二层筛选 — 商业价值

对每个项目评估7个维度（1-10分）：

| 维度 | 核心问题 |
|---|---|
| 需求真实性 | 有人真的愿意为此付钱吗？ |
| 获客可持续性 | 能稳定进客户吗？ |
| 交付成立性 | 能不能清晰、标准、稳定地交付？ |
| 利润空间 | 净利润够不够？ |
| 风险可控性 | 平台/法律/信用/经营风险可控吗？ |
| 资产沉淀能力 | 做完能留下什么长期价值？ |
| 壁垒潜力 | 别人容易抄走吗？ |

输出商业价值总评、最大优势、最大问题。

### 第4步：综合评级

根据两层得分给出综合评级：**S / A / B / C**

### 第5步：四类划分

将每个项目归入：
- **A类 长期主线型**：能形成核心能力、能沉淀资产、值得反复研究
- **B类 可测试型**：可小成本验证、能获取反馈和经验
- **C类 了解型**：值得懂不一定值得做
- **D类 排除型**：风险高、不稳定、长期价值差

### 第6步：验证建议

- 是否进入验证？是 / 否
- 更适合：长期研究 / 短测 / 观察 / 排除
- 核心理由
- 如果进入验证，第一轮重点测什么？

## 多项目横向比较

如果有多个项目，输出横向比较表，所有维度并列对比，最后给出推荐排序和理由。

## 输出格式

读取以下模板结构组织输出：
- `.cursor/模板/模板3-1：双层项目筛选表-标准版.md` — 单项目
- `.cursor/模板/模板3-1：双层项目筛选表-多项目横向比较版.md` — 多项目

## 完成后衔接

- 筛选出要验证的项目 → 提示说「帮我设计闭环」进入最小闭环设计
- 所有项目都不太行 → 提示回到收集阶段，多看几个案例再来
