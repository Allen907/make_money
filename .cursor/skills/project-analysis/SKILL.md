# 项目深度分析

基于《个人赚钱成长操作系统》第1篇「赚钱判断系统」的完整分析框架。详细原则、案例和判断标准见 [reference.md](reference.md)。

## 分析流程（9步）

收到用户的赚钱信息后，按以下9步严格执行：

### 第1步：信息类型判断

先判断这条信息属于哪种类型，不要急着评价：
- **具体项目**：有明确动作、产品/服务、收钱方式
- **赚钱逻辑**：更抽象，能跨项目复用的赚钱原理
- **放大手段**：不是项目本身，是做大的方法（矩阵、SEO、分销等）
- **包装话术**：听起来有道理但不够具体，容易制造幻觉
- **高风险玩法**：可能赚钱但风险大、不稳定、不可长期积累

如果信息包含多种类型，逐一拆出来标注。

### 第2步：赚钱本质判断

判断它本质赚的是什么钱。十种赚钱本质：
1. 卖产品 2. 卖服务 3. 卖知识 4. 卖流量 5. 赚差价
6. 赚渠道 7. 赚抽成 8. 赚品牌 9. 赚平台 10. 赚趋势

追问：主收入是什么？次收入是什么？哪一层是核心利润来源？这种钱长期稳不稳？

### 第3步：用户分析

- 它卖给谁？谁最容易买？谁最着急？谁最容易反复买？
- 用户处于什么阶段？当前最主要的问题是什么？

### 第4步：用户付费动因

从三大类判断：
- **省**：省时间、省精力、省试错、获得便利
- **得**：获得结果、身份感、稀缺资源、情绪价值、陪伴
- **解**：缓解焦虑、满足欲望

核心问题：用户真正买的是什么？是表面产品还是背后感受？

### 第5步：获客分析

判断主要获客方式（平台推荐/搜索/SEO/私域/广告/裂变/分销/老客复购/直播），回答四个必问问题：
1. 这个流量入口我拿不拿得到？
2. 这个入口稳不稳定？
3. 流量成本控不控制得住？
4. 流量能不能沉淀下来？

### 第6步：交付分析

判断交付类型（实物/数字产品/课程/服务/咨询/社群/陪跑/结果交付），检查交付成立性五件事：
1. 清不清晰 2. 能不能标准化 3. 成本高不高 4. 稳不稳定 5. 会不会翻车

### 第7步：商业模式归类与壁垒风险

归类到六种商业模式之一：技能服务型/知识内容型/产品卖货型/流量IP型/渠道撮合型/套利机会型

分析壁垒（技能/内容/信任/流量/渠道/供应链/工具效率/品牌/经验）和五类风险（平台/法律/信用/经营/个人）。

### 第8步：双层筛选评估

**第一层 执行可行性**：启动轻不轻/反馈快不快/成本高不高/与当前能力距离/能不能持续做30天

**第二层 商业价值**：需求真实吗/获客可持续吗/交付成立吗/利润够不够/风险可控吗/能沉淀资产吗/壁垒在哪里

### 第9步：综合结论

- **长期价值分类**：A类长期主线/B类可测试/C类了解/D类排除
- **最终判断**：可长期研究/可小规模测试/仅作了解/不建议做
- **如果要做，第一步应该是什么**

## 输出格式

读取 `.cursor/模板/模版1-1.赚钱项目分析卡-标准版.md` 的结构作为输出模板。将填充好的分析卡保存到 `案例分析/YYYY-MM-DD-项目名称.md`。

对信息不足的维度标注"信息不足，建议补充：（具体需要什么信息）"。

## 分析完成后

额外提供：
1. **项目四层拆解**：动作层 → 模式层 → 杠杆层 → 认知层
2. **与九种项目母型的对比**
3. **放大路径建议**（提价/提转化/提复购/标准化/矩阵化/渠道化，哪些适用）

然后输出阶段衔接建议：
- 结论为"值得做" → 提示说「帮我筛选」或「帮我设计闭环」
- 结论为"仅作了解" → 提示记录到案例收集表
- 结论为"不建议" → 提示记住问题出在哪里，练习判断力
