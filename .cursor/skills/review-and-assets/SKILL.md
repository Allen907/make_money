# 复盘与资产沉淀

基于《个人赚钱成长操作系统》第2篇第8章「复盘优化」+ 第3篇第1-3章。详细原则见 [reference.md](reference.md)。

## 核心原则

> 「复盘的真正意义不是总结今天干了啥。而是：把做过的事，转化成更好的判断和下一轮更有效的行动。」
> 「复盘最后一定要强迫自己选一个：继续/调整/放弃。」
> 「不要把复盘写成情绪日记。」

## 复盘流程（7步）

### 第1步：本周目标回顾
引导用户写清本周本来想验证或推进什么。

### 第2步：实际动作盘点
只写真实动作，不写大话。逐一列出做了什么、完成没有、结果是什么。

### 第3步：有效动作识别
哪个动作最有效？为什么有效？下周是否继续放大？

### 第4步：无效动作识别
哪个动作最无效？为什么无效？下周是否停止或调整？

### 第5步：问题归因（六层定位）
从六层中选一个主问题层：项目层 / 流量层 / 转化层 / 交付层 / 执行层 / 认知偏差层。
问：「本周最核心的问题在哪一层？」

### 第6步：学到了什么
- 本周真正看清了什么？
- 哪个判断被现实修正了？
- 用户真正关心的是什么？

### 第7步：下周决策
必须三选一，不能模糊：**继续 / 调整 / 放弃**
- 为什么做这个选择？
- 下周最关键的1-3个动作是什么？

> 「放弃并不丢人。前提是它来自清晰复盘，而不是情绪崩溃。」

## 高价值动作识别

复盘时额外检查动作价值分层：

**低价值动作示例**：无目的刷案例、不停改简介/排版/头像、学暂时用不上的工具、机械整理资料但不输出判断

**高价值动作示例**：看目标用户真实提问、和潜在用户聊天、记录用户咨询问题、设计报价和产品描述、发测试内容并设置咨询入口、试一次付费转化、把用户问题整理成FAQ、把交付动作整理成SOP

引导用户回答：
- 最近最耗时间的低价值动作是什么？
- 最应该增加的高价值动作是什么？
- 下周优先做哪一个高价值动作？

## 资产沉淀检查

复盘完成后，逐一检查九类资产：
1. 内容资产（可复用内容、高反馈表达、FAQ、选题方向）
2. 用户资产（接触到真实用户、留下联系方式、用户理解更清晰）
3. 信任资产（获得案例、评价/反馈、用户对我更信任）
4. 产品/模板资产（形成产品雏形、模板/SOP、交付流程）
5. 方法资产（更新了判断框架、验证了有效路径、明确了无效路径）
6. 其他资产（SEO资产、渠道资产、私域资产）

资产浪费复盘：
- 本轮最大的资产浪费是什么？
- 哪些明明做了但没有留下来？
- 下轮准备怎么补沉淀？

## 输出格式

读取以下模板结构组织输出：
- `.cursor/模板/模板8：每周复盘模板.md`
- `.cursor/模板/模板10：资产沉淀表.md`
- `.cursor/模板/模板13：高价值动作识别表.md`

## 完成后衔接

- 如果决策为"继续" → 提示说「帮我设计闭环」优化下一轮验证
- 如果决策为"调整" → 提示说「帮我分析」重新评估方向
- 如果决策为"放弃" → 提示记录到资产沉淀表，保留学到的东西，然后回到收集阶段
